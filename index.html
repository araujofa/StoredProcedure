Stored Procedure

Nós fizemos esse site na intenção de compartilhar conhecimento sobre Stored Procedure.
Vamos abordar sobre o que é o stored procedure, para que serve e como usar.

Stored Procedure em português procedimento armazenado é um código SQL preparado para que o código possa ser reutilizado repetidamente. Portanto, se você tiver uma consulta SQL que você escreve repetidamente, salve-a como um procedimento armazenado e chame-a para executá-la. Você também pode passar parâmetros para um procedimento armazenado, para que o procedimento armazenado possa agir com base nos valores dos parâmetros que são passados.




A Stored Procedure serve quando:
Temos várias aplicações escritas em diferentes linguagens, ou rodam em plataformas diferentes, porém executam a mesma função.
Queremos diminuir o tamanho do código ao encurtar partes que se repetem. Dentro de uma Stored Procedure, você pode escrever comandos DML (Linguagem de manipulação de dados = Insert, Delete, Update, Merge) , DDL (Linguagem de definição de dados– Create, Drop, Alter) e DQL (Linguagem de consulta de dados = Select). O SQL Server armazena em cache os planos de execução das consultas e, normalmente, os reutiliza em execuções subsequentes da procedure para economizar tempo, CPU e recursos de memória associados à otimização das consultas.




As Stored Procedures são criadas de formas simples e incrivelmente lógicas em relação a repetição de trechos complexos de código.

As Stored Procedures fornecem muitos benefícios em comparação à implementação da lógica de negócios no aplicativo. Eles encapsulam a lógica para permitir a reutilização e ocultam a complexidade. É muito mais fácil aplicar alterações a um procedimento armazenado com um comando como ALTER PROCEDURE simples em comparação à implementação de alterações no aplicativo.

Para chamar uma stored procedure normalmente tem um comando EXECUTE ou CALL, ou DO ou RUN, dependendo do banco de dados. É um comando SQL como outro qualquer. Chamou, executa o que precisa.

* Note que o que tem @ são os parâmetros que podem ser passados para ela. O resto são comando da linguagem, que obviamente pode executar qualquer comando SQL. Mas não só. É uma linguagem de programação como outra qualquer (ao contrário do SQL que não é uma linguagem de programação).
